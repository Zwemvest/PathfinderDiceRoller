  - Everything from compact PLUS:

  - Shift note if shifted (e.g., "Failure -> Success (nat 20)").

**History page** (`src/routes/history/+page.svelte`):
- Replace the current stub content entirely.
- Use Dexie `liveQuery` with `$effect` pattern (from RESEARCH.md) to reactively load rolls:
  ```typescript
  import { liveQuery } from 'dexie';
  import { db } from '$lib/db';
  import type { RollSnapshot } from '$lib/db';
  import { browser } from '$app/environment';

  let rolls = $state<RollSnapshot[]>([]);

  $effect(() => {
    if (!browser) return; // guard against SSR
    const obs = liveQuery(() =>
      db.rollHistory.orderBy('rolledAt').reverse().limit(100).toArray()
    );
    const sub = obs.subscribe({ next: (data) => { rolls = data as RollSnapshot[]; } });
    return () => sub.unsubscribe();
  });
  ```
- **Layout:**
  - Header: "Roll History" title + "Clear History" button (top-right, destructive red styling).
  - Scrollable list of HistoryEntry components.
  - Chat-log feel: use CSS `flex-direction: column-reverse` on the list container so newest rolls appear at bottom and scrolling goes up for older rolls. Since liveQuery returns newest-first (`.reverse()`), the column-reverse will flip them so newest is at bottom visually.
  - Auto-scroll to bottom on new roll: use `$effect` that watches `rolls.length` and scrolls the container to bottom.
  - Empty state: if no rolls, show a friendly "No rolls yet" message with dice icon.
- **Clear History:**
  - "Clear History" button triggers a confirmation dialog (can use a simple native `confirm()` or a custom modal).
  - On confirm: call `clearRollHistory()` from `$lib/db/roll-helpers`.
  - After clear, liveQuery automatically updates the list to empty.

**Styling:**
- Use existing Tailwind theme tokens.
- History entries have subtle separator lines between them.
- Expanded entries have a slightly different background (bg-surface-overlay).
- DoS badge colors match RollResultCard exactly (DEGREE_COLORS).
