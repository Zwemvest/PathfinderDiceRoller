    - Test 4: getRollHistory() returns entries ordered by rolledAt descending (newest first)

1. In `src/lib/db/index.ts`:
   - Add `RollSnapshot` interface replacing the old `RollHistoryEntry` concept:
     ```typescript
     export interface RollSnapshot {
       id?: number;
       rolledAt: Date;
       characterName: string;    // "Knurvik" or "Free-form"
       label: string;            // "Athletics", "Fortitude", "2d8+4"
       rollType: 'skill' | 'save' | 'perception' | 'initiative' | 'free-form';
       notation: string;         // "1d20+15"
       dieResults: number[];     // raw die faces
       naturalDie: number;       // first die (d20 face for checks)
       modifierTotal: number;    // net modifier
       total: number;            // final roll total
       keptModifiers: string;    // JSON-serialized kept[] for breakdown display
       dc: number | null;
       degree: string | null;    // DegreeOfSuccess string or null
       shifted: boolean;
       shiftDirection: string | null; // 'up' | 'down' | null
     }
     ```
   - Add `version(3)` block to AppDatabase constructor. Keep same index signature `'++id, rolledAt'` for rollHistory — new fields are stored automatically without indexing. No `.upgrade()` callback needed (old entries with missing fields are acceptable since they'll be cleared or render gracefully with nullish checks).
   - Update the `rollHistory` table type from `Table<RollHistoryEntry>` to `Table<RollSnapshot>`.
   - Keep the old `RollHistoryEntry` interface marked as `@deprecated` for backwards compat.
   - Export `RollSnapshot` from the module.

2. Create `src/lib/db/roll-helpers.ts`:
   - `saveRoll(snapshot: Omit<RollSnapshot, 'id'>): Promise<number>` — checks count, if >= 100 deletes oldest entries (orderBy('rolledAt').limit(count - 99).primaryKeys() then bulkDelete), then adds the new entry.
   - `getRollHistory(limit = 100): Promise<RollSnapshot[]>` — returns `db.rollHistory.orderBy('rolledAt').reverse().limit(limit).toArray()`.
   - `clearRollHistory(): Promise<void>` — calls `db.rollHistory.clear()`.

3. Create `src/lib/db/roll.browser.test.ts` with the behavior tests above. Use `beforeEach` to clear the rollHistory table. Import from `$lib/db` and `$lib/db/roll-helpers`.

CRITICAL: Do NOT index `keptModifiers`, `dieResults`, or other complex fields in the Dexie schema string — only `++id, rolledAt` is needed. Dexie stores complex objects via structured clone without indexing.

CRITICAL: The file extension for the state module in Task 2 is `.svelte.ts` (not `.ts`) — Svelte 5 runes only work in `.svelte.ts` files.
