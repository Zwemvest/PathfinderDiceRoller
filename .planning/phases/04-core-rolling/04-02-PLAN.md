**PreRollDialog.svelte** (`src/lib/components/rolling/PreRollDialog.svelte`):
- Use native `<dialog>` element with `showModal()` / `close()`.
- Bind `dialogEl` with `$state<HTMLDialogElement | null>(null)`.
- `$effect` watches `dialogState.open` — if true, `dialogEl.showModal()`; if false, `dialogEl.close()`.
- Listen to `onclose` event to sync `dialogState.open = false` (NOT `oncancel` alone — see pitfall 3 in RESEARCH.md).
- Display content:
  - Roll label (e.g., "Athletics") as header
  - Base modifier displayed as "+15" (read-only)
  - Modifier toggles: clone `PRESET_MODIFIERS` into local `$state` array. Render as pill-shaped toggleable chip buttons (button elements with active/inactive styling — large tap targets for mobile). Clicking a chip toggles its `enabled` field.
  - DC entry field: number input, optional (leave empty for no DC).
  - For initiative rollType: show a `<select>` dropdown listing all skills (passed as optional prop `availableSkills: NormalizedSkill[]`). Default selection is "Perception". When a skill is picked, update `dialogState.baseModifier` to that skill's total and `dialogState.label` to "Initiative (Stealth)" etc.
  - "Roll" button: on click, execute the roll:
    1. `const activated = localModifiers.filter(m => m.enabled)`
    2. `const resolved = resolveModifiers(activated)`
    3. `const notation = '1d20+' + (dialogState.baseModifier + resolved.total)` (handle negative totals correctly — if sum is negative, notation becomes `1d20-N`)
    4. `const roll = rollExpression(notation)`
    5. `const dc = dcInput || null`
    6. `const degree = dc !== null ? computeDegree(roll.total, dc, roll.naturalDie) : null`
    7. Build RollSnapshot, call `saveRoll(snapshot)`, call `setLastRoll(snapshot)`, call `closePreRollDialog()`
  - CRITICAL: For free-form rolls (rollType === 'free-form'), the notation comes from the dice tray input, NOT from `1d20+modifier`. Do not call computeDegree unless the notation starts with `1d20` (see pitfall 5 in RESEARCH.md).

**RollResultCard.svelte** (`src/lib/components/rolling/RollResultCard.svelte`):
- Reads `rollState` from the state module.
- If `rollState.snapshot === null`, render nothing (empty/hidden).
- **Compact view** (default, `rollState.expanded === false`):
  - Roll label (e.g., "Athletics"), total in bold, natural d20 highlighted/bold, DoS badge if degree is set.
  - DoS badge: inline colored badge using `DEGREE_COLORS` — Gold = crit success, Green = success, Orange = failure, Red = crit failure. Text: degree name with hyphens replaced by spaces, uppercased.
  - If `shifted === true`, show shift note as small subtitle: e.g., "Failure -> Success (nat 20)".
  - Character name in small muted text.
  - Tap anywhere on the card to toggle `rollState.expanded`.
- **Expanded view** (`rollState.expanded === true`):
  - Everything from compact view PLUS:
  - Full modifier breakdown: parse `keptModifiers` (JSON string) back to `ModifierEntry[]`, group by type (status, circumstance, item, untyped), display each with label and signed value.
  - Notation string and individual die results.
  - DC value if set.
  - Post-roll DC entry: a number input that, when changed, recalculates degree of success using `computeDegree(snapshot.total, newDC, snapshot.naturalDie)` and updates the display (but does NOT re-save to DB — this is a "what if" check).

**DiceTray.svelte** (`src/lib/components/rolling/DiceTray.svelte`):
- Collapsed by default: show a thin bar with a dice icon and "Free Roll" text. Tap to expand.
- Expanded: text input for dice expression + "Roll" button.
- On Roll:
  1. Call `openPreRollDialog(expressionText, 0, 'free-form', activeCharacterName || 'Free-form')` to open the pre-roll dialog with modifier toggles and DC field available.
  2. The PreRollDialog handles the actual rolling (for free-form, it uses the expression text as notation instead of building `1d20+N`).
  - Alternative simpler approach: If the expression is simple (no modifier toggles wanted), roll directly:
    1. `rollExpression(expressionText)`
    2. Build RollSnapshot with rollType 'free-form', characterName 'Free-form' or active character name
    3. `saveRoll(snapshot)`, `setLastRoll(snapshot)`
  - Support both paths: direct roll button for quick rolls, and a modifier toggle icon that opens the full pre-roll dialog for the expression.
- CRITICAL: For non-d20 expressions, set `dc: null, degree: null, shifted: false, shiftDirection: null` in the snapshot (see pitfall 5).
