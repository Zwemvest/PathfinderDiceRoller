  <name>Task 2: Implement Foundry parser (GREEN)</name>
  <files>src/lib/parsers/foundry.ts</files>
  <behavior>
    - parseFoundry(json: unknown) returns NormalizedCharacter
    - Ability scores reconstructed from ancestry boosts/flaws, background boosts, class key ability, and level boosts (system.build.attributes.boosts)
    - Saves derived from class item savingThrows ranks + ability mods (fort=CON, ref=DEX, will=WIS)
    - Skills from system.skills merged with class/background trained skills as rank-1 fallbacks
    - Perception from class item perception rank + WIS mod
    - Weapons from weapon items with attack bonus = ability mod + proficiency bonus + potency + bonus.value
    - HP = (classHp * level) + ancestryHp + (CON_mod * level)
    - Spellcasting from spellcastingEntry + spell items (empty array for non-casters)
    - Feats from feat items with category mapping
    - Throws ImportError on missing required fields
  </behavior>
  <action>
    Create `src/lib/parsers/foundry.ts` implementing `parseFoundry(json: unknown): NormalizedCharacter`.

    Follow the exact reconstruction algorithms from 02-RESEARCH.md:

    1. **Validation:** Verify json has `name`, `type === "character"`, `system.details.level.value`. Use `require()` helper from types.ts.

    2. **Ability reconstruction:** Start all at 10. Apply ancestry boosts (selected or single-value fixed), ancestry flaws, background boosts, class key ability boost, then level boosts from `system.build.attributes.boosts`. Convert to modifiers via `Math.floor((score - 10) / 2)`.

    3. **Class item extraction:** Find `items.find(i => i.type === 'class')`. Extract savingThrows ranks, perception rank, attack proficiencies, trainedSkills, HP per level.

    4. **Save computation:** For each save, total = ability_mod + profBonus(rank). Foundry rank is 0-4, profBonus = rank === 0 ? 0 : level + rank * 2.

    5. **Skill computation:** Merge system.skills with class/background trained skills. For each skill, total = ability_mod + profBonus(rank). Store proficiencyRank in 0-4 scale.

    6. **Weapon parsing:** For each weapon item, compute attackBonus = abilityMod + profBonus(weaponProfRank) + potency + bonus.value. Use STR unless finesse trait AND DEX > STR. Dice count = damage.dice + strikingRune.

    7. **HP:** maxHp = (class.hp * level) + ancestry.hp + (CON_mod * level). currentHp from system.attributes.hp.value.

    8. **Spells:** Collect spellcastingEntry items for tradition/DC/attack. Collect spell items grouped by level. Non-casters get empty arrays.

    9. **Feats:** Map feat items, translating category strings (e.g., "classfeature" stays, "Ancestry Feat" maps from Foundry's category naming).

    CRITICAL: Handle the edge cases from 02-RESEARCH.md pitfalls:
    - Never read system.abilities (always null)
    - boost.selected may be null for free boosts — skip if unselected
    - system.skills may be empty {} — that's valid
    - Weapon finesse: use DEX only if trait includes "finesse" AND dex_mod > str_mod
  </action>

  <name>Task 3: Implement Pathbuilder parser (GREEN)</name>
  <files>src/lib/parsers/pathbuilder.ts</files>
  <behavior>
    - parsePathbuilder(json: unknown) returns NormalizedCharacter
    - Abilities from build.abilities (pre-computed scores), converted to modifiers
    - Skills use profBonus = rank === 0 ? 0 : level + rank (Pathbuilder scale), plus mods[TitleCase].["Item Bonus"]
    - Saves use same prof formula + ability mod + item bonus from mods
    - Weapons use pre-computed attack bonus, striking rune mapping for dice count
    - Spells from spellCasters array with spell attack and DC computed
    - HP matches formula: classhp*level + ancestryhp + bonushp + bonushpPerLevel*level + conMod*level
    - Feats from build.feats tuples [name, null, category, level, ...]
    - proficiencyRank stored normalized to 0-4 Foundry scale (divide Pathbuilder rank by 2)
  </behavior>
  <action>
    Create `src/lib/parsers/pathbuilder.ts` implementing `parsePathbuilder(json: unknown): NormalizedCharacter`.

    Pathbuilder is the easier parser — most values are pre-computed.

    1. **Validation:** Verify json has `success === true` and `build` object. Use `require()` helper.

    2. **Identity:** Direct reads from build.name, build.class, build.level, build.ancestry, build.heritage, build.background, build.keyability.

    3. **Abilities:** Read build.abilities.{str,dex,con,int,wis,cha} (these are SCORES not modifiers). Convert to modifiers via `Math.floor((score - 10) / 2)`.

    4. **Skills:** For each standard PF2e skill, read build.proficiencies[skillName]. Pathbuilder proficiency bonus = rank === 0 ? 0 : level + rank (rank already includes *2 factor). Add item bonus from build.mods[TitleCaseSkillName]?.["Item Bonus"] ?? 0. Total = abilityMod + profBonus + itemBonus. Store proficiencyRank normalized to 0-4 scale (Pathbuilder rank / 2).

    5. **Lore skills:** Read build.lores[] tuples. Ability = INT. Same prof formula.

    6. **Saves:** For each save, read build.proficiencies[saveName]. Total = abilityMod + profBonus + mods item bonus. Fort=CON, Ref=DEX, Will=WIS.

    7. **Perception:** build.proficiencies.perception. Total = WIS_mod + profBonus + mods["Perception"]?.["Item Bonus"] ?? 0.

    8. **Weapons:** Map build.weapons[]. Use pre-computed `attack` field as attackBonus. Map `str` field to strikingRune (""=0, "striking"=1, "greaterStriking"=2, "majorStriking"=3). Dice count from striking mapping. Include extraDamage array.

    9. **Spells:** Map build.spellCasters[]. Compute spellAttack = profBonus(proficiency) + abilityMod. spellDC = 10 + spellAttack. Map spells and prepared arrays.

    10. **HP:** Formula from research: classhp*level + ancestryhp + bonushp + bonushpPerLevel*level + conMod*level.

    11. **Feats:** Map build.feats tuples. Category at index 2, normalize to FeatCategory type.

    12. **AC:** build.acTotal.acTotal direct read.

    13. **Class DC:** 10 + profBonus(build.proficiencies.classDC) + keyAbilityMod.

    CRITICAL: Pathbuilder `weapon.str` is the striking rune string, NOT the STR ability. Name variables carefully.
  </action>
