    2. **ImportZone.svelte** — The main import component. Props: none (self-contained with DB access).
       State machine with $state():
       - `idle`: Shows format picker + file input button + drag-drop zone
       - `parsing`: Shows spinner
       - `success`: Shows confirmation card — "Imported {name} (Level {level} {class})" with accent border
       - `error`: Shows error message (red border) with the ImportError message
       - `confirm-reimport`: Shows changes summary when name matches existing character

       File handling:
       - `<input type="file" accept=".json">` hidden, triggered by button click
       - Drag-drop zone with `ondragover` (prevent default, highlight border) and `ondrop` (read file)
       - Use `file.text()` (Promise-based) to read file content
       - `JSON.parse()` wrapped in try/catch (catch shows "Invalid JSON file" error)
       - Call `parseFoundry()` or `parsePathbuilder()` based on selected format
       - On success: check `findCharacterByName()` for existing character
         - If exists: show confirm-reimport state with diff summary (level change, new feats, etc.)
         - If new: save directly via `saveCharacter()`, set as active, show success
       - On ImportError: show error state with the error message

       Re-import changes summary: Compare old and new NormalizedCharacter on key fields (level, class, feats count, skills count, weapons count) and list what changed.

       Props: `character: StoredCharacter`, `isActive: boolean`, `onSelect: () => void`, `onDelete: () => void`
