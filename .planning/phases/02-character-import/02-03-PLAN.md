  <files>src/lib/components/character/CollapsibleSection.svelte, src/lib/components/character/CharacterDashboard.svelte, src/lib/components/character/SkillsSection.svelte, src/lib/components/character/SavesSection.svelte, src/lib/components/character/AttacksSection.svelte, src/lib/components/character/SpellsSection.svelte, src/lib/components/character/FeatsSection.svelte</files>
  <action>
    Build all 7 character display components using Svelte 5 runes and Tailwind CSS v4. All components receive data via $props() — no internal DB access.

    1. **CollapsibleSection.svelte** — Reusable wrapper.
       Props: `title: string`, `defaultOpen: boolean = true`, `count?: number`, children snippet.
       Header row with title, optional count badge, chevron icon that rotates. Clicking header toggles content visibility. Use $state() for open/closed. Animate with Tailwind transition.

    2. **CharacterDashboard.svelte** — Top overview bar.
       Props: `character: NormalizedCharacter`
       Layout:
       - Row 1: **Name** (large), Class + Level badge (e.g. "Monk 1")
       - Row 2: Ancestry / Heritage / Background in subtle text
       - Row 3: Ability modifier chips — 6 compact chips showing "+4 STR", "+2 DEX" etc. Use accent color for the key ability. Per locked decision: modifier only, no raw scores.
       - Row 4: HP bar (currentHp / maxHp), AC badge, Perception badge (+N), Hero Points indicator
       - Mobile-first: stacks vertically on narrow screens, spreads on wider

    3. **SkillsSection.svelte** — Wrapped in CollapsibleSection.
       Props: `skills: NormalizedSkill[]`
       Per locked decision: compact 2-column grid. Each cell shows skill label + total modifier (e.g. "Acrobatics +7"). Lore skills included at the bottom. Sort alphabetically. Untrained skills (rank 0) shown in muted text. Trained+ in normal text. In Phase 4 these become tappable — for now, display only.

    4. **SavesSection.svelte** — Wrapped in CollapsibleSection.
       Props: `saves: NormalizedSaves`
       Three prominently displayed saves: Fortitude, Reflex, Will — each showing the total modifier. Horizontal row layout with equal spacing.

    5. **AttacksSection.svelte** — Wrapped in CollapsibleSection.
       Props: `weapons: NormalizedWeapon[]`
       Per locked decision: compact display showing weapon display string (e.g. "+1 Striking Staff +15"). Each weapon as a row/card. Show damage expression in subtle text below (e.g. "2d8+4 B"). ExtraDamage entries listed if present. In Phase 5 these become tappable with MAP.

    6. **SpellsSection.svelte** — Wrapped in CollapsibleSection.
       Props: `spellcasters: NormalizedSpellcaster[]`, `totalFocusPoints: number`
       Per locked decision: spell names grouped by spell level. If multiple spellcaster entries (like Kairos), show each tradition as a sub-section. For each spell level, show slot count (if applicable) and spell names. Show "No spells" message if spellcasters array is empty.

    7. **FeatsSection.svelte** — Wrapped in CollapsibleSection.
       Props: `feats: NormalizedFeat[]`
       Per locked decision: grouped by category (Class Feat, Ancestry Feat, Skill Feat, General Feat, etc.). Each group as a sub-header with feat names listed below. Names only — no mechanical effects (deferred to v2).

    DESIGN PRINCIPLES:
    - Information-dense but not cluttered (per user's specific request — "looking at this between combat rounds on your phone")
    - Use Tailwind theme tokens from app.css (--color-surface-*, --color-text-*, --color-accent)
    - Font sizes: compact but readable on mobile (text-sm for most, text-base for important values)
    - Modifiers always show sign: "+4" not "4", "-1" not "1"
    - All sections default to expanded except Feats (collapsed by default — least frequently referenced)

    CRITICAL: Use Svelte 5 $props() for all component inputs. NO `export let`.
    CRITICAL: Use {@render children()} in CollapsibleSection, not <slot>.
  </action>
